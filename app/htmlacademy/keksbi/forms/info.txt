Подготавливаем блок отзывов

В этой части Кексби-курса мы добавим в разметку блок с формой вопросов, в которой будут текстовые поля ввода, «галочка», раскрывающийся
список и
кнопка отправки. Формы вы изучали в курсе «Знакомство с формами».

Раздел формы обратной связи разметим дивом с классом feedback, добавим внутрь него заголовок второго уровня и поясняющий текст.
 Поясняющий текст — это полноценное предложение, поэтому для его разметки можно использовать тег p с классом feedback-tip.
---------------------------------------------------------------------
Добавляем форму

Теперь добавим саму форму и зададим ей необходимые атрибуты action и method.

На сервере Кекса обработка форм производится по адресу /keksby-mail, поэтому его и задаём в action.

Методом отправки будет post, так как сообщения могут быть достаточно большими и в адресную строку
браузера могут не поместиться (метод get отправляет данные через адресную строку).
---------------------------------------------------------------------
Поле «Имя»

Теперь, когда у нас есть форма, начнём заполнять её.

Каждое поле ввода и подпись к нему будем оборачивать в блок с классом feedback-form-group. Это нужно, чтобы в дальнейшем расположить
 поля как на макете.

В макете подписи к полям располагаются на отдельной строчке, поэтому мы не будем оборачивать текст подписи и поле внутрь одного тега label.
 Вместо этого будем подписи и поля располагать рядом друг с другом и связывать их с помощью атрибутов for и id.

Разметим поле для ввода имени. Это простое текстовое поле. Главное придумать для него хорошее имя и идентификатор. Вполне подойдёт fullname.
------------------------------------------------------------------------
Поле «Телефон»

Поле для ввода телефона разметим по аналогии с полем имени. Название и идентификатор — phone.

Иногда используют одинаковые названия и идентификаторы полей, а иногда разные. Например, для всех
идентификаторов можно добавлять суффикс «field», например:

<input type="text" name="lastname" id="lastname-field">
Оба подхода имеют право на существование. Главное, чтобы значения id были уникальными на странице.
-------------------------------------------------------------------------
Поле «Email»

Поле для ввода электронной почты разметим по аналогии с телефоном, изменив название и идентификатор на email.

Позже, на этапе стилизации, мы расставим поля для ввода телефона и почты в две колонки.
-----------------------------------------------------------------------------
Список с темами обращений

Теперь добавим в форму раскрывающийся список с темами обращений.

Его разметка аналогична первым трём полям формы: используется контейнер .feedback-form-group, подпись и поле связываются с помощью for и id.
Имя и идентификатор списка — topic.
-------------------------------------------------------------------------------
Темы обращений

Добавим содержимое в раскрывающийся список из прошлого задания. Там будут перечислены самые частые темы обращений.
 В зависимости от выбранной темы скрипт-обработчик формы будет отправлять её разным сотрудникам в корпорации Кекса. Например, вопросы
  про возврат товара отправляются бухгалтеру, а жалобы лично Кексу.
Возможные варианты в раскрывающемся списке задаются с помощью тегов option. Напомним, что значение варианта содержится в
 атрибуте value тега option, а текст-описание варианта внутри тега option. Текст-описание варианта и значения могут не совпадать.

В нашей форме значения вариантов будут заданы максимально просто, с помощью чисел, чтобы сервер мог их с лёгкостью обработать.
------------------------------------------------------------
Поле для сообщения

Последнее поле в форме будет содержать текст обращения пользователя. Сообщение может быть объёмным, поэтому давайте сделаем это поле
 ввода многострочным <textarea>.

Разметка аналогична первым трём полям. Название и идентификатор поля — message.
-----------------------------------------------------------
Отступы в форме

Отлично, форма готова. Теперь добавим для блоков .feedback-form-group небольшие внутренние отступы, чтобы поля не «прилипали» друг к другу.

На этом шаге разметка главной страницы завершена и вас ожидает первая часть итогового испытания, в которой вы тоже будете заниматься разметкой
 (оно будет опубликовано очень скоро).

Вот архив с текущим состоянием магазина Кексби.

В следующих частях Кексби-курса мы приступим к стилизации полученной страницы.
